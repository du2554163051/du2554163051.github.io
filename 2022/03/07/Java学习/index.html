    类和常用方法

    1、Object类

    • Object类是所有类的顶级父类，是所有类的根，所有类都直接或间接继承Object类

    • (String) toString()：使用System.out.println()打印对象时，调用该方法返回对象的地址。子类可以重写此方法实现输出其他的值。

    • (boolean) equals(Object obj)：比较两个引用是否指向同一个对象，也就是两个引用的地址是否相同；如果比较基本数据类型类型，则比较值是否相同。如果想要比较两个对象的内容是否相同，需要重写equals方法

    • (Object) clone()：它实现对象中各个属性的复制，但它的可见范围是protected的，所以实体类使用克隆的前提是：

      • 实现Cloneable接口，这是一个标记接口，自身没有方法。
      • 覆盖clone()方法，可见性提升为public
      • 深克隆与浅克隆
        • 浅克隆：被复制对象的所有值属性都含有与原来对象的相同，而所有的对象引用属性仍然指向原来的对象。
        • 深克隆：在浅拷贝的基础上，所有引用其他对象的变量也进行了clone，并指向被复制过的新对象

    • (int) hashCode()：获取对象的哈希值，此哈希值为对象涉及集合操作时提供参考，也可以将这个哈希值理解为地址值

    2、String类

    特点

    • 被final修饰，不能被继承

    • 不可变对象，每次对字符串的修改都会生成新的字符串对象

    • 内部维护了一个char数组，用于存储数据

    • 内部维护了一个字符串常量池，可以直接使用字面量来创建对象，字面量创建的对象都会缓存到常量池中，以达到复用的效果。如果创建时使用new关键字赋值时，创建的字符则不会缓存到常量池，每次都会重新创建一次

      String s1 = "hello"+"world";
String s2 = "hello";
String s3 = s2 + "world";
// 字符串在使用 “+” 做拼接时，如果两边都是字面量，编译优化结果仍然放在常量池，如果“+”两边有变量参与，会创建新的字符串对象，放在堆空间中
System.out.println(s1 == s3) //false

    方法

    • toUpperCase() ：将字符串转为大写并返回
    • toLowerCase() ：将字符串转为小写并返回
    • endsWith(String x) ：是否以x结尾，返回值为boolean
    • startsWith(int start) ：是否以start开始，返回值为boolean
    • charAt(x) ： 获取指定位置x的字符，返回值为char
    • indexOf(x) ： 正向获取指定字符串x在原字符串中的位置，如果没找到返回-1，返回值为int
    • lastIndexOf(x) : 反向获取指定字符串x在原字符串中的位置，如果没找到返回-1 返回值为int
    • split(x) : 根据指定的正则表达式x来拆分字符串，返回值为String[]
    • substring(x,y)： 截取字符串中从x到y的字符串
    • replace(x,y) : 使用y字符串替换x字符串
    • replaceAll() : 使用正则表达式匹配替换字符串
    • trim()： 去掉字符串两端的空格
    • concat() ： 字符串拼接
    • contains() ： 是否包含子字符串，返回值boolean
    • equalsIgnoreCase() ： 忽略大小写比较，返回值boolean

    3、StringBuffer类

    特点

    • 可变字符串，StringBuffer&&StringBuider
    • StringBuffer是可变字符串类型，对其操作不会产生新的字符串对象，操作的都是其本身
    • StringBuffer内部维护着char数组，默认长度是16，有自动扩容的机制，当容量不够时，会自动扩容，最大能扩容到Integer.MAX_VALUE-8
    • StringBuffer没有重写equals方法
    • StringBuffer是早期的实现，是线程安全的。StringBuilder是后期的实现，线程不安全
    • 当涉及字符串的频繁修改时，应当选择StringBuffer提升效率

    方法

    • append(): 追加字符串操作

      String s = null;
String s1 = null;
System.out.println(s+s1)//"nullnull"

/**
* +运算中在做字符串拼接时，如果有变量参与
* 1.创建一个StringBuffer对象

    • insert(int index,String x) : 向指定位置index添加字符串x

    • delete(int start,int end) : 删除指定区间start-end的字符串

    • replace(int start,int end,String x) : 将指定区间start-end的字符串替换成x

    7、Random随机数类

    特点

    • Random类中实现的随机算法是伪随机，也就是有规则的随机。在进行随机时，随机算法的起源数字称为种子数(seed)，在种子数的基础上进行一定的变换，从而产生需要的随机数字

    方法

    • public boolean nextBoolean() : 该方法的作用是生成一个随机的boolean值，生成true和false的值几率相等，也就是都是50%的几率
    • public double nextDouble() : 该方法的作用是生成一个随机的double值，数值介于[0,1.0)之间
    • public int nextInt() : 该方法的作用是生成一个随机的int值，该值介于int的区间，也就是-231到231-1之间
    • public int nextInt(int n) : 该方法的作用是生成一个随机的int值，该值介于[0,n)的区间，也就是0到n之间的随机int值
    • public void setSeed(long seed) : 该方法的作用是重新设置Random对象中的种子数。设置完种子数以后的Random对象和相同种子数使用new关键字创建出的Random对象相同

    Collection接口

    • 代表所有集合的顶级父接口，定义了集合中常见的操作方法
    • 他是所有集合的根，分别有List(有序集)、Set(无序集)、Queue(队列)三类实现
    • 集合可以看成存储数据的容器，他可以存储不同数据类型的值，但在实际应用中通常只存储一种数据类型的数据

    方法

    • public boolean add() : 添加元素
    • public boolean addAll() : 集合合并
    • public boolean remove() : 删除元素
    • public boolean removeAll() : 根据子集来删除
    • public boolean contains() : 是否包含指定元素
    • public boolean containsAll() : 是否包含指定子集合
    • public int size() : 查看集合的大小
    • public boolean isEmpty() : 是否是一个空集合
    • public void clear() : 清空集合

    List接口

    特点

    • List集合是Collection的子接口，具备Collection定义的常用操作
    • List代表有序集，元素是有序的(存取顺序)
    • 可以通过下标来随机访问元素，下标从0开始到size-1结束，使用下标访问元素时必须是合理的下标，否则会抛出IndexOutOfBoundsException
    • 允许重复元素存在
    • List是一种线性顺序的数据结构

    方法

    • 由于List接口是Collection的子接口，以下只介绍List独有的方法和重载的方法
    • T get(int index) : 获取对应位置index的数据，用法与数组相似，从零开始计数
    • void add(int index,T,element) : 在指定位置index添加数据element，并且从index开始的所有元素以此后移
    • T set(int index,T element) : 在指定位置index的数据替换为element

    ArrayList类

    特点

    • ArrayList是List接口的具体实现类，数组实现
    • 底层维护着一个Object类型数组，用于存储数据，数组默认长度为10
    • 这个数组有自己的扩容机制，当容量不够时，自动扩容为原来的1.5倍，最大能扩容到Integer.MAX_VALUE
    • ArrayList是List接口的后期的实现，非线程安全的，可以通过Collections辅助工具转为线程安全的集合
    • 可以调用有参构造为ArrayList指定初始容量
    • 因为是数组实现，查询效率高、增删元素效率低

    LinkedList类

    特点

    • LinkedList 是List接口的实现类
    • 底层是链表实现，而且是双向链表，内部也维护着一个下标
    • 因为是链表实现，查询效率低、增删元素效率高，查询头尾节点效率高
    • LinkedList还实现了Queue接口，也可以将LinkedList当成队列来使用
    • LinkedList也是非线程安全的实现
    • LinkedList在存储元素时，会将元素转为一个节点，节点包含以下3个部分
    • 第一部分(pre): 上一个节点的引用
    • 第二部分(E): 元素本身
    • 第三部分(next): 下一个节点的引用
    • 链表中只有一个数据时，这个数据既是头节点也是尾节点

    Vector类

    特点

    • Vector是List接口的实现类
    • 底层也是数组实现，数组的默认长度是10，有自己的扩容机制，如果没有指定增量那么将扩容为原容量的2倍。如果指定了容量则扩容为原容量+增量
    • Vector是List的早期实现类，是线程安全的实现，故效率较低

    Stack类

    特点

    • 是Vector类的子类，内部也是数组实现
    • 栈是一种先进后出的数据结构
    • 压栈：将元素压入栈顶
    • 栈顶：栈中最后一个添加的元素
    • 弹栈：获取栈顶元素，并删除
    • 栈底：栈中最先添加的yuans

    方法

    • public E push(E item) : 把项压入堆栈顶部
    • public E pop() : 移除堆栈顶部的对象，并作为此函数的值返回该对象。
    • public E peek() : 查看堆栈顶部的对象，但不从堆栈中移除它
    • public int search(Object o) : 返回对象在堆栈中的位置，以 1 为基数
    • public boolean empty() : 测试堆栈是否为空

    HashSet类

    特点

    • Set接口的实现类，底层采用hash算法来存储数据
    • HashSet实质上是一个没有value值的HashMap

    HashMap类

    特点

    • 采用数组+链表的形式来存储数据，数组的默认长度是16
    • 在存储元素时，先根据元素的hashCode值计算元素在数组中存放的位置
      • 如果这个位置没有元素，直接存储
      • 如果有元素，依次与该位置上现有元素equals比较
        • 如果equals为true的情况，则替换原有的值
        • 如果equals为false的情况，以链表形式存储
    • HashMap的扩容机制
      • 数组的默认长度是16，当HashMap数组所用超过75%时，会自动扩容为原长度的两倍
      • 扩容后现有元素全部进行rehash(重排列)
      • 当数组的长度超过64，且链表长度超过8时，该链表会转化为红黑树结构
    • 在构建一个HashMap时，可以指定初始容量，当指定初始容量后，HashMap会重新根据指定的初始容量来计算一个新的初始容量，新的初始容量一定是一个比初定容量大2的次幂，之所以这样做的目的为了保证(n - 1) & hash算法恒成立，计算的结果一定保证在数组下标
    • HashMap是非线程安全的实现
    • 元素在集合中操作一定要保证内容相同的对象，hashCode值相同，equals相同。内容不同的对象，hashcode一定不同，equals也不同，否则不能保证HashSet/HashMap的效率
    • HashSet集合中，元素的修改不会导致元素的重排列，只有在扩容时才会重排列，应尽量避免hashSet集合中元素的修改
    • HashMap优化
      • 初始容量问题，加载因子
      • 尽量避免链表的长度不超过8
      • 结合现有数据的大小
